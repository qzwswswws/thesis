### 2.1 系统整体性质

从仓库命名、板级配置与源代码实现可以确认，该系统并非单纯的脑电采集板，而是一个集成了脑电采集与基础生理参数监测的便携式多模态终端，当前工程名称为 `NeuraMeter V1 EEG+HR Board`。

可以直接确认的要点如下：

- 固件基于 `Zephyr RTOS`
- 上位机基于 `Qt6 + QML + Qt Bluetooth`

## 3. 已确认的硬件架构信息

### 3.1 主控与板级平台

从板级文件可确认：

- 板卡名称：`NeuraMeter V1 EEG+HR Board`
- 芯片平台：`nRF52832`
- RAM：`64 KB`
- Flash：`512 KB`
- 工具链：`Zephyr`

这一信息可以作为第三章硬件平台选型与嵌入式软件环境的基础描述。

### 3.2 脑电采集链路

从固件实现可以确认，脑电信号链路大致为：

1. 电极信号进入 `KS1092` 前端
2. `KS1092` 的增益由 MCU 通过 `SPI` 配置
3. 模拟输出送入 `nRF52832 SAADC`
4. MCU 完成双通道采样、滤波、缓冲与 BLE 打包发送
5. 上位机接收数据并完成实时显示与频谱分析

这一点很重要：现有代码更像是“`KS1092 + MCU ADC`”的模拟前端方案，而不是“独立数字脑电 AFE 直接输出数字帧”的方案。第三章写作时应避免把 `KS1092` 误写成直接完成数字化采集的芯片，除非后续原理图显示并非如此。

### 3.3 已确认的接口与引脚信息

根据板级设备树和驱动代码，可以先整理出如下接口映射：

- EEG ADC 通道
  - `CH0 -> AIN0 (P0.02)`
  - `CH1 -> AIN2 (P0.04)`
- KS1092 控制接口
  - `SPI1 SCK -> P0.18`
  - `SPI1 MOSI -> P0.11`
  - `SPI1 MISO -> P0.17`
  - `KS1092 CS -> P0.15`
  - `KS1092 RST -> P0.19`
  - `KS1092 EN -> P0.20`
  - `KS1092 LDF -> P0.23`
- 电源/按键相关
  - `POWER_OFF -> P0.16`
  - `DETKEY -> P0.12`
- UART 调试口
  - `TX -> P0.06`
  - `RX -> P0.08`
- MAX30102 / MAX30101
  - `I2C SDA -> P0.30`
  - `I2C SCL -> P0.31`
  - 注释中还提到 `INT -> P0.29`
- 板载指示灯
  - `LED -> P0.24`

这些内容已经足够在第三章画出一张“系统接口分配表”。

## 4. 固件架构与流程信息整合

### 4.1 固件软件框架

固件采用 `Zephyr RTOS`，核心模块至少包括：

- `main.c`：系统启动、蓝牙初始化、采样主循环
- `eeg_acquisition.c`：双通道 ADC 采样与环形缓冲
- `ks1092.c`：KS1092 的 SPI 配置与状态读取
- `iir_filter.c`：工频陷波、低通/带通滤波
- `max30102_collector.c`：心率血氧采集线程
- `hr_spo2_algo.c`：MAX30102 数据处理算法

第三章写作时，可以把固件描述为“事件驱动与周期采样相结合的模块化固件架构”。

### 4.2 EEG 采样流程

代码显示 EEG 采样流程为：

1. 初始化 SAADC 与双通道输入
2. 设置采样率 `500 Hz`
3. 使用 `2 ms` 周期定时器触发采样任务
4. 在工作队列中执行 ADC 读取，避免阻塞定时器回调
5. 将采样结果写入环形缓冲区
6. 主循环从缓冲区读取样本并执行数字滤波
7. 根据当前 MTU 将多个样本打包为 NUS 二进制帧发送

这是一个非常适合写入论文的实现细节，因为它能体现嵌入式实时性设计，而不是单纯“能采到数据”。

### 4.3 采样与缓冲参数

目前源码中可明确读出的参数包括：

- EEG 采样率：`500 Hz`
- 主程序初始化缓冲深度：`1024`
- 模块支持的最大缓冲深度：`2048`
- 每个样本结构：
  - `ch0: int16`
  - `ch1: int16`
  - `timestamp: uint32`
- 主循环处理块大小：`30`

第三章中可以把这部分写成“低功耗无线采集条件下的数据缓冲机制设计”。

### 4.4 增益与前端控制

`KS1092` 当前实现支持两通道独立增益控制，支持的可选增益值包括：

- `360`
- `540`
- `680`
- `720`
- `1020`
- `1080`
- `1360`
- `1440`

主程序默认将双通道增益都设置为 `1440`。同时，上位机也支持动态发命令修改增益。

这一点对论文很有帮助，因为它说明系统不是固定参数采集板，而是具备一定可配置性的实验平台。

### 4.5 数字滤波能力

固件内已经实现了多种滤波模式：

- 关闭滤波
- `50 Hz` 陷波
- `~100 Hz` 低通
- 组合滤波
- `0-45 Hz` 带通

并且两通道可分别配置。默认配置为“双通道组合滤波启用”。

这部分完全可以写成第三章中的“工频陷波算法实现”和“可配置数字滤波策略”。

### 4.6 电极状态检测

系统支持 `LDF / lead-off` 检测，并通过 GPIO 读入脱落状态。代码注释表明：

- `LDF 高电平 -> 电极脱落`
- `LDF 低电平 -> 电极连接正常`

这说明硬件并不只是“采数据”，还具备基础的连接状态自检能力，这一点值得在论文中强调。

### 4.7 电源与关机逻辑

固件中还实现了较完整的上电保持与按键关机逻辑：

- `P0.16` 用于电源保持
- `P0.12` 用于按键检测
- 按键保持约 `1000 ms` 触发关机

如果后续 EDA 中能找到对应电源自锁电路，这部分可以写成完整的便携设备电源管理设计。

## 5. 无线通信与协议信息整合

### 5.1 BLE 传输方式

仓库采用 `BLE Peripheral + Nordic UART Service`。

可以直接确认的配置包括：

- 设备名称：`NeuraMeter`
- 最大连接数：`1`
- 已开启 NUS 服务
- 已配置较大的 ACL 数据长度与 `L2CAP MTU=247`

这说明系统不是简单串口透传，而是面向较大吞吐量做过 BLE 参数优化。

### 5.2 EEG 二进制帧格式

上位机与固件对 EEG 数据帧的定义是一致的，可以直接写入论文协议部分：

- `magic: 1 byte`，固定为 `0xE0`
- `seq: 2 bytes`
- `count: 1 byte`
- `samples: count * 8 bytes`

其中每个样本为：

- `ch0: int16`
- `ch1: int16`
- `timestamp: uint32`

这是一份非常有价值的论文素材，因为可以把它写成“脑电数据无线传输帧结构设计图”。

### 5.3 批量打包策略

固件根据当前 BLE MTU 动态调整每帧携带样本数：

- 默认 MTU 初值：`23`
- 连接后会接收 MTU 更新回调
- 每帧样本数会按 `(mtu - 3) / 8` 估算
- 最大批量上限：`20`

这说明系统考虑到了 BLE 实际传输效率，而不是固定逐样本发送。

### 5.4 控制与查询命令

固件当前支持的文本指令至少包括：

- `GAIN ch1 ch2`
- `QUERY GAIN`
- `FILTER channel mode enable`
- `QUERY FILTER`
- `QUERY STATUS`
- `QUERY STATS`
- `QUERY HR`
- `CLEAR BUFFER`
- `RESET KS1092`

对应的返回消息为 `ACK ...` 或 `ERR ...`。

这说明上位机并非只是被动显示，而是具备硬件配置、状态查询与故障恢复功能。

### 6.4 实时信号处理策略

上位机并不是简单画波形，而是已经包含了较完整的在线处理逻辑：

- 维护 `4000` 点环形缓冲

- FFT 点数：`2048`
- 将 ADC 值换算为微伏量级显示
- 计算相对频带能量分布
- 支持信号有效性检测
- 支持短时伪迹检测与保护区屏蔽

当前频段划分大致为：

- Delta：`3.5-4.5 Hz`

- Alpha：`8-13 Hz`
- Beta：`14-30 Hz`

注意：这里的 Delta 范围与传统文献定义略有差异，第三章可以如实写为“本系统实现中的工程化频段划分”，而不要直接说成标准医学定义。
