# 给 Codex 的第 3 章起草提示词

你现在需要起草论文第 3 章“脑电采集硬件系统设计与实现”。

## 起草前必读
1. `00_AI_Management/Prompt_Library/Chapter_Writing_Style_Guide.md`
2. `00_AI_Management/Prompt_Library/Writing_Quality_Standards.md`
3. `00_AI_Management/Prompt_Library/Hardware_Chapter_Writing_Protocol.md`
4. `00_AI_Management/Output_Drafts/Chap03_Hardware_Info_Integration_20260329.md`
5. `00_AI_Management/Output_Drafts/Chap03_Material_Inventory_20260329.md`
6. `02_Source_Material/03_Hardware_Workbench/README.md`

## 本章核心目标
第 3 章应写成学位论文中的“系统设计与实现”章节，而不是工程操作手册、源码说明书或产品宣传页。

正文需要围绕以下问题展开：
- 本系统要解决什么问题，为什么需要自研便携式双通道脑电采集硬件。
- 系统为何采用“脑电前端 + 主控与无线传输 + 上位机”的总体架构。
- 各模块分别承担什么功能，它们如何形成完整采集闭环。
- 关键器件与接口为何这样选取，这样设计如何满足实时采集、无线传输与后续实验需求。
- 该系统设计如何为后续算法验证和系统实验提供硬件基础。

## 特别加入的写作边界

### 上位机在第 3 章与第 5 章的分工
第 3 章可以写上位机，但只写其在系统闭环中的设计与实现角色，包括：
- BLE 连接与数据解析链路；
- 实时波形显示与频带能量可视化；
- 参数配置与设备状态查询功能；
- 作为硬件系统配套软件所承担的人机交互职责。

第 3 章不展开写以下内容：
- 上位机的实验表现、实时性结果、稳定性结果；
- 延迟、吞吐、丢包、长期运行表现；
- 上位机如何支撑具体实验任务并得到何种效果。

这些更适合在第 5 章写成系统验证与实验结果。

一句话原则：
- 第 3 章回答“上位机是什么、如何实现、为什么需要它”；
- 第 5 章回答“上位机在实验中表现如何、支撑效果怎样”。

## 写作红线
以下内容默认不应直接写入正文主段落：
- 按步骤操作式语言；
- 编译命令、点击按钮说明；
- 大段源码名、函数名、日志名；
- 临时版本号、仓库内部命名；
- 大段寄存器说明和引脚堆砌；
- “这里可以看到”“点击后”“代码里实现了”这类工程口语。

## 本章特别要防止的问题

### 1. 防止写成工程手册
不要只写：
- 模块有哪些；
- 接口接到了哪里；
- 命令格式是什么。

而要写：
- 为什么需要这些模块；
- 为什么采用这样的链路和协议；
- 这种设计如何满足双通道脑电采集、无线传输和可视化交互的系统目标。

### 2. 防止写成器件罗列
不要把 BOM 逐项抄进正文。
正文只重点分析：
- KS1092 脑电采集前端；
- nRF52832 主控与 BLE SoC；
- 供电与充电相关器件；
- 晶振、接口和关键外围网络。

### 3. 防止写成源码翻译
固件部分应转写为：
- 采样流程；
- 缓冲机制；
- 数字滤波策略；
- 分帧传输逻辑；
- 状态查询与参数配置机制。

而不是逐个文件解释谁调用谁。

### 4. 防止过度下结论
只有当源码、原理图、BOM、PCB 图等材料共同支持时，才能写成确定事实。
对于尚未完全确认的内容，使用保守表述。

## 推荐段落结构
每段优先遵循以下逻辑：
1. 先说明该模块或设计要解决的问题；
2. 再说明采用的实现方案；
3. 然后解释这样设计的技术理由；
4. 最后回到系统目标，说明其对整机和后续实验的意义。

## 本章应突出但不过度展开的内容
- 系统总体架构；
- 脑电前端电路；
- 主控与无线传输电路；
- 供电与接口设计；
- 固件采样与 BLE 数据链路；
- 上位机的实时显示与参数配置功能。

## 图表使用要求
优先考虑以下图片进入正文：
- 前端局部原理图；
- 主控局部原理图；
- PCB 三维结构图；
- 上位机界面截图。

插图后必须补充分析，说明该图支持了什么设计结论，不能只配一句图注结束。

## 语言要求
- 使用正式、克制、学术化表达；
- 避免产品说明式和宣传式语气；
- 避免夸张词；
- 优先使用“表明”“说明”“支撑”“满足”“用于实现”等稳健表述。

## 执行要求
在真正修改 `chap03.tex` 时：
- 以完整起草为主，不写成碎片化提纲；
- 在保持章节主结构清晰的前提下，可以为了学术表达适度调整小节标题；
- 保持与第 1、2、4 章一致的学位论文语气；
- 优先保证逻辑完整、术语正式、段落连贯，再补足细节。
