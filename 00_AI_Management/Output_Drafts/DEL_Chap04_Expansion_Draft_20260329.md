# 第4章扩写可行性评估与正式化改写草稿

现有材料**足以支撑**第 4 章再扩写一轮，而且不仅能扩写，还可以把当前章节里偏“过程记录”的表述进一步收束成更像学位论文终稿的叙事。

但这个“足以支撑”有边界：

- **足够支撑的部分**
  - 通道退化实验的设计逻辑、结果表述与深入讨论

  - 低通道结构探索的负结果总结

  - 把低通道新结构写成“已经形成最终定型模型”
  - 把低通道结构探索写成“显著优于 baseline 的最终方案”
  - 在扩展验证部分加入正式的复杂度、FLOPs、ONNX、量化和 RK3568 实测数据
  - 把 EdgeMIFormer 在少通道条件下写成已经完整闭环验证

换句话说，现有材料完全可以把第 4 章从“已有主体 + 一组结果补丁”扩成“已有研究基础 + 统一协议下的扩展验证 + 工程边界讨论”的完整章节；但还不足以把它扩成“少通道新模型已经完全定稿”的版本。

## 3. 当前第4章最需要调整的不是数据，而是表述

### 3.1 不建议继续出现“已发表小论文”

正式论文里不宜频繁直接说“已发表小论文”“小论文基础上……”。  
更稳妥的学术写法应改为：

- `前期研究工作`
- `前期已完成的研究基础`
- `既有研究结果`
- `前期算法设计与验证工作`

推荐替换关系：

| 当前表述 | 建议替换 |
| --- | --- |
| 已发表小论文 | 前期研究工作 / 前期已完成研究 |
| 小论文基础上的补充实验与结果分析 | 前期研究基础上的扩展验证与结果分析 |
| 小论文主体 | 前期算法主线 / 既有模型主线 |

### 3.2 不建议继续使用 `lowchannel_v1` 这类内部开发命名

这类命名适合代码仓和实验记录，不适合论文正文。

推荐改成下面这种正式命名：

| 内部命名 | 论文建议命名 |
| --- | --- |
| `lowchannel_v1` | 低通道原生结构重构方案 |
| `Batch 1` | 分类头重构方案 |
| `Batch 2` | 差分表征增强方案 |
| `KD pilot` | 知识蒸馏补偿验证 |

这样改的好处是：

- 读者不用理解你的开发过程编号
- 每个方案的研究意图一眼就能看懂
- 更符合正式论文的表述习惯

### 3.3 “补充实验”可进一步学术化

“补充实验”本身不算错，但在第 4 章里会略显口语、略像阶段记录。

更合适的替代词：

- `扩展验证`
- `扩展研究`
- `附加验证`
- `进一步验证`

其中最推荐的是：

- 标题层面用 `扩展验证`
- 段落层面用 `进一步验证` 或 `衍生验证`

## 4. 第4章可扩写的方向

在不引入新实验的前提下，现有材料至少还能支撑 4 个方向的扩写。

### 4.1 扩写方向一：把实验协议写得更完整

当前第 4 章已经提到数据集和部分设置，但还可以再补强：

- 为什么采用单被试训练、单被试测试
- 为什么统一用 `A0xT` 训练、`A0xE` 测试
- 为什么要固定标准化方式
- 为什么少通道验证必须在同协议下进行

这部分扩写的价值很高，因为它能让后面的比较更“合法”。

### 4.2 扩写方向二：把通道退化的分析写得更深

现有退化结果已经足够支撑更深入的讨论，例如：

- 为什么 `22 -> 8` 导的下降相对可控
- 为什么 `8 -> 3` 导后仍保留一定研究价值
- 为什么 `3 -> 2` 导会出现更明显的断崖
- 为什么这不是“模型失效”，而是“空间判别信息被压缩后的物理边界”

这部分是最值得扩的，因为它直接提升第 4 章的分析厚度。

### 4.3 扩写方向三：把“双导二分类”写成工程路线收束

当前双导二分类结果很有价值，可以进一步写得更完整：

- 不只是说“准确率提升了”
- 还要说：
  - 为什么双导条件下二分类更贴近硬件展示目标
  - 为什么它更符合 C3/C4 的侧化特征
  - 为什么它是“任务目标收束”，不是“单纯降难度”

这部分扩写后，第 4 章会更自然地和后续系统章节对接。

### 4.4 扩写方向四：把探索线写成“研究边界”

当前探索线材料完全够写一个更像研究论文的段落群：

- 低通道原生结构重构方案未形成稳定收益
- 分类头重构方案未稳定超过 baseline
- 差分表征增强方案改善有限
- 知识蒸馏补偿验证显示出有限但一致的正向趋势

这部分不是凑字数，而是在回答：

`在当前边界条件下，哪些方向值得保留，哪些方向不宜再继续加码？`

这会让第 4 章更完整，也更像一个真正做过研究筛选的章节。
