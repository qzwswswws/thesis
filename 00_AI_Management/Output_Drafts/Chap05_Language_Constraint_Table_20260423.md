# 第 5 章语言约束与替换表（2026-04-23）

## 1. 用途

本文档用于把第 5 章中的“实验记录语言”转换为“论文正文语言”。  
它主要解决四类问题：

1. 把内部实验名翻成正文口径。
2. 把口语、批注和工程记录腔删掉。
3. 把问题句、路径句、证据句和边界句分开。
4. 把图表解释和结论力度压到合适范围。

---

## 2. 总原则

### 2.1 问题句与路径句分离

- 问题句只写“要解决什么”。
- 路径句只写“本章如何论证”。
- 结论句只写“当前证据能支撑什么”。

禁止把三者混成一句。

### 2.2 先证据，后解释，再收束

第 5 章的大多数段落应遵循：

`先给对象和口径 -> 再给关键数字或图表 -> 再解释意义 -> 最后收住边界`

### 2.3 内部代号不直接进入正文

凡是目录名、脚本名、变量名、运行编号、调试标签，默认都不能直接进入正文。  
正文只保留自然语言解释后的表达。

## 4. 禁用表达与替换建议

### 4.1 口语和项目记录腔

| 禁用表达 | 替换建议 |
| --- | --- |
| `试跑` | `真实在线实验` / `真实在线运行` |
| `联调` | `联合验证` / `链路验证` |
| `做出来了` | `已经实现` / `已经形成` |
| `这里可以看出` | `结果表明` / `图 X 显示` |
| `这样说明了` | `由此说明` |
| `我们又做了` | `进一步补充了` / `随后补充了` |

### 4.2 批注腔

| 禁用表达 | 替换建议 |
| --- | --- |
| `需要说明的是` | 直接删掉，改为事实陈述 |
| `需要指出的是` | 直接写边界句 |
| `这里要注意` | `应注意` 或直接写限制条件 |
| `不宜写成` | 放在质控文档里，不进入正文 |

### 4.3 过强结论

| 禁用表达 | 替换建议 |
| --- | --- |
| `已经充分证明` | `结果表明` / `可作为初步依据` |
| `已经稳定成立` | `已经表现出初步支撑` |
| `显著优于所有方法` | `更值得保留` / `在当前协议下表现更优` |
| `已经完成完整闭环` | `已经形成训练-反馈联动机制` |

### 4.4 偷懒指代

| 禁用表达 | 替换建议 |
| --- | --- |
| `这一点` | 直接复述所指对象 |
| `上述内容` | 直接写出前一判断 |
| `前一种情况` | 直接写具体策略或具体口径 |
| `后者` | 若句子变长，改成完整对象名 |

## 5. 第 5 章常用句型模板

### 5.1 问题句模板

`本节关注的问题是：________________________________。`

用途：只交代本节要回答什么，不提前写结论。

### 5.2 路径句模板

`围绕这一问题，本文先________，再________。`

用途：只写论证路径，不写结论。

### 5.3 证据句模板

`在________条件下，A 为 ________，高于/低于 B 的 ________；该差异主要体现在________。`

用途：先对象，后数字，再指出差异所在。

### 5.4 图表解释句模板

`图 X 给出的不是________，而是________。`

`图 X 主要用于回答“________”这一问题。`

用途：防止把图解释成它本来不承担的命题。

### 5.5 边界句模板

`不过，该结果的证据边界仍收在________，不能进一步写成________。`

用途：防止从局部结果跳到整体结论。

### 5.6 收束句模板

`因此，本节更稳妥的结论是：________。`

`由此，本节将________定位为________。`

## 6. 第 5 章中的专门约束

### 6.1 在线结果的三种口径必须分开

| 口径 | 正文怎么写 | 不能怎么写 |
| --- | --- | --- |
| 总体汇总 | `总体在线判断准确率` | 直接拿来代表完整会话水平 |
| 完整会话 | `一次 20 个试次的完整会话达到……` | 与短时运行混写 |
| 短时局部运行 | `一次仅含 8 个有效判断的短时运行达到……` | 写成稳定可用性证明 |

### 6.2 显著性检验必须说明三件事

1. 用了什么检验。
2. 零假设是什么。
3. 显著性结果支撑的是哪一层判断。

例如：

`对应的显著性检验采用 20,000 次循环移位置换检验，其零假设为保持各方法自身序列不变、仅打乱方法之间的时间对齐关系；若观测统计量仍显著高于该零分布，则说明共享困难结构并非随机对齐造成。`

### 6.3 图注要短，解释放正文

- 图注只写对象和作用，不写长段方法解释。
- 详细说明放正文首段或图后段。
- 图注不承担“证明链”，正文承担。

## 7. 改写前后的判断标准

### 7.1 下面这种写法不合格

`本章试图解决的是先用伪在线找一个比较好的方法，然后再放到真实在线里验证，最后说明系统已经可以用了。`

问题：

1. 问题和路径混写。
2. `比较好的方法` 过空。
3. `可以用了` 结论越界。

### 7.2 更合格的写法

`本章试图解决的问题是：在低通道采集、有限校准样本和真实运行链路约束下，低通道运动想象脑电解码系统如何从离线研究进一步走向在线交互，并获得具有实际参考意义的初步验证。`

`围绕这一问题，本文先通过公开数据集上的伪在线评估筛选可保留的快速适配路径，再在真实在线实验中考察训练-反馈联动机制及其初步验证结果。`

---

## 8. 最后检查表

每次改完一个小节，按下面七条检查：

1. 问题句、路径句、结论句是否混在了一句里。
2. 是否仍有内部实验名直接进入正文。
3. 是否把局部高分写成整体稳定结论。
4. 是否只报数字、不解释数字说明什么。
5. 是否只讲图好看、不讲图回答什么问题。
6. 是否有口语和批注腔残留。
7. 是否有一句明确的收束句。
