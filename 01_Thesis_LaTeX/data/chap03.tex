\chapter{脑电采集硬件系统设计与实现}
\label{cha:hardware}

为满足低通道脑机接口系统对便携化、低功耗和无线化采集平台的需求，本文设计并实现了一套面向双通道脑电采集的嵌入式硬件系统。该系统以脑电前端采集链路为核心，结合低功耗主控芯片、蓝牙无线传输链路以及配套上位机软件，形成了从信号采集、参数配置、数据传输到实时可视化显示的完整闭环。与仅面向离线算法研究的数据集环境不同，本章关注的是面向真实采集场景的系统实现问题，即如何在有限板级资源与无线带宽约束下完成双通道脑电信号的稳定获取，并为后续算法验证与系统实验提供可复用的硬件平台。

本章将从总体架构、关键电路、固件流程以及系统联调验证四个层面展开论述。其中，总体架构部分重点说明系统模块划分及其协同关系；电路设计部分围绕脑电采集前端、主控与无线传输链路以及供电接口设计进行分析；固件部分说明采样、滤波、缓冲与分帧传输的实现逻辑；最后结合当前已完成的联调结果，对系统的闭环可用性与实现特征进行分析。

\section{硬件系统总体架构设计}

\subsection{模块功能分工}

本文所设计的硬件系统由脑电采集前端模块、主控与无线传输模块、供电与接口模块以及配套上位机软件组成，其核心目标是在保证系统便携性和可配置性的前提下，实现双通道脑电信号的连续采集与实时显示。系统的主数据链路为：电极采集到的模拟脑电信号首先进入前端采集芯片完成模拟调理与增益控制，随后由主控芯片进行模数转换、缓存管理与数字滤波，并通过 BLE 链路发送至上位机，由上位机完成帧解析、波形显示、频带能量估计和参数配置。除脑电主链路外，系统还预留了基于 I\textsuperscript{2}C 总线的基础生理信号扩展接口，用于实现心率与血氧等辅助生理信息的采集与显示。

从系统级角度看，该平台并非单一的采样板卡，而是一套围绕双通道脑电采集构建的完整实验终端。一方面，前端采集芯片负责弱脑电信号的模拟耦合与放大，为后续数字化提供稳定输入；另一方面，低功耗主控芯片同时承担参数控制、数据打包和无线通信职能，使系统在板级面积、功耗和复杂度之间取得平衡。各模块分工如表\ref{tab:c3_module_split}所示。

\subsection{关键器件选型与方案比较}

在总体架构确定之后，脑电前端与主控无线平台的选型直接决定了系统的通道冗余、板级复杂度、功耗水平以及后续固件实现边界。对本文所面向的双导便携式原型而言，器件选型并非单纯追求参数越高越好，而是需要在连续采样能力、无线传输能力、体积与功耗约束、以及后续算法联调的可扩展性之间取得平衡。因此，本节分别围绕脑电前端芯片和主控无线芯片，对当前方案与典型替代方案进行比较，并说明本文最终采用 KS1092 与 nRF52832 组合的原因。

\subsubsection{脑电前端芯片方案比较}

在脑电前端芯片方面，当前学术研究和原型系统中较常见的一类方案是以 ADS1299 为代表的高集成度多通道生物电采集前端。ADS1299 面向脑电与其他生物电测量设计，集成了多通道 PGA、24 bit 同步采样 ADC、内部参考和振荡器，并通过 SPI 直接输出数字化结果，适合构建高导联、高保真度的研究级采集平台\cite{ads1299_ti}。这类器件的优势在于模拟链路和数字化链路高度集成，便于获得统一的采样时钟和较高的幅值分辨率；但对于本文所需的双导、低功耗、小板级面积原型而言，其多通道能力和高集成度同时也意味着明显的资源冗余。

与之相比，本文实际采用的 KS1092 更接近双通道便携式脑电前端的实现需求。结合本地数据手册、BOM 和原理图可以确认，KS1092 在当前系统中承担的是双通道模拟前端角色，支持通过 SPI 进行增益与状态配置，并提供导联脱落检测信号；其模拟输出随后进入主控侧 SAADC 完成数字化\cite{ks109x_datasheet}。这种“模拟前端 + 主控侧 ADC”的路径在转换精度和前端集成度上不如 ADS1299 一类多通道专用 AFE，但能够显著降低当前双导系统的通道冗余和板级实现复杂度，同时为主控侧滤波、缓存与打包策略保留更大的实现灵活性。两类方案的主要差异如表\ref{tab:c3_afe_compare}所示。

综合来看，本文并未将 KS1092 视为对 ADS1299 的“全面替代”，而是将其视为更符合当前任务边界的工程取舍：前者更适合围绕双导连续采样、板级紧凑实现和后续 BLE 回传构建原型，后者则更适合高导联、高精度和更强同步数字化需求的研究平台。对本研究而言，前端选型的关键不是追求多通道极限性能，而是为后续双导任务、无线链路和算法联调提供稳定且可控的硬件基础。

\subsubsection{主控与无线平台方案比较}

主控芯片的选型需要同时考虑四类要求：其一，能够承担双通道连续采样与前端参数控制；其二，能够在电池供电条件下维持 BLE 持续连接；其三，具备足够的片上资源支撑 Zephyr 固件、缓存管理和协议栈；其四，尽量降低外接无线模块和跨芯片互连带来的面积与功耗开销。围绕这些要求，nRF52832、STM32WB55 与 ESP32 可以视为三类有代表性的主控路线。

nRF52832 是一类面向低功耗蓝牙应用的高集成 SoC，集成了 64 MHz Cortex-M4F 内核、BLE 无线收发器、片上 ADC 以及 512 KB Flash 和 64 KB RAM，适合构建以 BLE 为主链路的小型近体设备\cite{nrf52832_nordic}。STM32WB55 则提供了更高的存储资源和更丰富的无线能力，其双核结构支持 BLE 与 802.15.4 等协议，并具备 1 MB Flash、256 KB RAM 和 12 bit ADC，适合需要更大协议栈空间或多协议扩展的无线节点\cite{stm32wb55_st}。ESP32 具有双核最高 240 MHz、Wi-Fi 与 Bluetooth/BLE 组合通信以及较丰富的片上外设，更适合侧重联网能力或本地通用计算的 IoT 设备\cite{esp32_espressif}。但对持续脑电采样终端而言，Wi-Fi 并非必要能力，额外的无线与功耗管理复杂度反而会提高系统实现负担。三类主控路线的比较如表\ref{tab:c3_mcu_compare}所示。

综合比较后，本文选择 nRF52832 作为主控与无线传输核心。其原因并不在于该芯片在所有维度上优于其他方案，而在于它更符合当前双导脑电终端的阶段性目标：一方面，BLE SoC 与片上 SAADC 的集成方式有助于缩短主数据链路、降低板级面积并减少跨芯片连接；另一方面，其 512 KB Flash 与 64 KB RAM 已足以支撑当前固件、缓存与协议栈需求，使系统能在较低功耗和较小体积下实现稳定采样与无线回传。相比之下，若后续研究重点转向更复杂的边缘端本地推理或多协议无线扩展，则 STM32WB55 或更高算力平台会成为更值得考虑的选择。

\subsection{接口协议设计}

为适应低功耗无线采集场景，本文采用 Nordic UART Service（NUS）作为设备与上位机之间的通信承载协议。该方案的优点在于实现复杂度较低、兼容性较好，并可同时支持连续数据上传与文本命令交互。考虑到系统既需要持续发送脑电数据帧，又需要支持参数配置、状态查询与错误回执，协议设计采用了“二进制连续数据帧 + ASCII 文本命令/回执”的混合方式。其中，脑电波形数据采用紧凑的二进制帧结构发送，以提高有效负载利用率；命令控制与状态回执则使用 ASCII 文本消息，便于上位机解析与调试。

在脑电数据链路中，每帧数据以固定魔数 `0xE0` 作为起始标识，用于与文本回执消息区分；帧头中进一步包含序列号与样本数量字段，随后按顺序附带多个双通道采样点。每个采样点包含两个通道的原始 ADC 值以及设备侧时间戳。该结构既能满足上位机对连续波形的实时显示需求，又保留了基础时序信息，便于后续进行丢包检测与链路分析。其帧结构如表\ref{tab:c3_ble_frame}所示。

\subsection{配套上位机软件闭环设计}

在本系统中，上位机并非独立于硬件之外的附属调试工具，而是采集闭环中的重要组成部分。其主要职责包括完成 BLE 设备发现与连接、订阅数据通知、区分二进制脑电帧与文本回执消息、实时显示双通道波形、估计频带能量分布以及向硬件下发增益和滤波配置命令。通过将这些功能整合到统一界面中，系统能够在采集过程中同时完成状态观测与参数控制，从而提高联调效率。

图\ref{fig:c3_ui}给出了配套上位机界面示意。界面左侧为设备连接与状态区，可显示设备连接状态与导联状态；中部为双通道原始脑电实时波形；下方进一步显示频带能量分布和部分生理参数信息；右侧则提供后端增益调节与滤波模式设置功能。该界面布局的设计思路并不是追求复杂交互，而是围绕“采集状态是否正常、波形是否连续、参数是否可配置”这三个核心问题服务于硬件系统本身。因此，在第 3 章中，上位机主要被视为硬件平台的配套实现部分，而其在实验中的实际表现与效果分析将放在后续章节中进一步展开。

\section{核心硬件电路设计}

系统的硬件电路设计围绕“弱脑电信号稳定获取、低功耗主控处理以及无线链路可靠传输”三项要求展开。考虑到脑电信号幅值较低且易受环境干扰影响，前端电路需要兼顾输入保护、导联连接稳定性与可调增益能力；而主控部分不仅要承担模数转换与通信任务，还需要在有限资源条件下支撑蓝牙协议栈和实时采样任务。基于上述考虑，本文采用 KS1092 作为脑电前端采集芯片，采用 nRF52832 作为主控与无线传输核心，并配合电源管理与外围接口电路形成完整硬件平台。

\subsection{脑电采集前端电路设计}

脑电采集前端电路是整套系统中最关键的模拟链路，其作用在于将来自电极接口的微弱脑电信号稳定耦合至后级采样链路。图\ref{fig:c3_front_end}给出了脑电前端局部原理图。从电路结构可以看出，系统通过多针接口引出双通道输入与参考/驱动信号，前端输入侧配置了保护网络，以增强实际采集条件下对静电与外部瞬态干扰的承受能力。双通道脑电信号分别进入 KS1092 的差分输入端，形成两条并行采集链路。

在功能实现上，KS1092 负责完成前端增益配置、导联脱落检测以及与主控之间的配置交互。芯片的配置接口通过 `SCLK`、`SDI`、`SDO` 与 `CS` 与主控侧相连，`RESET` 与 `EN` 引脚用于上电初始化与工作状态控制，`LDF` 信号则用于向主控反馈电极接触状态。该设计使脑电前端不再是固定参数的黑盒模块，而是一个可被主控动态配置并具备基础自检能力的采集单元。结合固件实现，系统支持多个离散增益档位配置，从而可针对不同被试、不同电极接触条件和不同实验任务进行前端调节。

需要指出的是，当前系统采用的是“模拟前端 + 主控侧 ADC 采样”的实现路径，即 KS1092 完成前端模拟调理后，将输出信号送入主控芯片的模数转换通道，而不是直接将数字化脑电数据从前端芯片侧输出。这一路径在硬件资源和板级复杂度上更为可控，同时也为系统保留了在主控侧执行滤波、缓存与打包策略的灵活性。对于本研究所关注的低通道便携式系统而言，这种设计兼顾了实现复杂度与实验可扩展性。

\subsection{主控与无线传输电路设计}

主控与无线传输部分采用 nRF52832 作为核心芯片，负责完成双通道采样控制、前端参数配置、BLE 通信以及部分系统状态管理。图\ref{fig:c3_main_control}给出了主控局部原理图。该芯片集成 ARM Cortex-M4 内核、片上 SAADC 与 BLE 无线通信能力，适合构建低功耗、小体积的近体式采集终端。与“前端采样芯片 + 外部 MCU + 外部蓝牙模块”的分立式方案相比，采用单芯片主控可显著简化板级互连关系，降低系统面积与功耗开销。

从接口分配上看，系统将 `P0.02` 和 `P0.04` 配置为脑电采样输入通道，用于接收来自前端的两路模拟输出；`P0.18`、`P0.11`、`P0.17` 与 `P0.15` 分别构成与前端芯片的 SPI 控制链路；`P0.19`、`P0.20` 与 `P0.23` 则负责前端复位、使能与导联状态检测。此外，电路中配置了 `32\,\text{MHz}` 高频晶振和 `32.768\,\text{kHz}` 低频晶振，分别服务于主系统运行与低功耗时基需求；右侧射频匹配网络与天线连接则构成了 BLE 无线传输的硬件基础。

该部分电路设计的关键在于让同一主控芯片同时承担“采样入口”和“无线出口”两个角色。前者要求其能够以固定采样周期读取双通道 ADC 数据并及时写入缓存，后者要求其能够在蓝牙链路中完成 MTU 协商、通知发送和状态回执。nRF52832 的选型使这两项要求能够在单板平台上统一实现，从而为后续的软件架构设计提供了稳定硬件基础。

\subsection{供电与接口设计}

为了满足便携式使用需求，系统在供电与接口部分采用了电池供电、Type-C 充电与板级稳压相结合的方案。根据器件清单与板级布局信息，系统包含 Type-C 接口、充电管理芯片、稳压器件、电池接口以及与按键相关的电源保持逻辑。该设计允许设备脱离外部实验台式电源独立运行，并为后续移动场景下的脑电采集提供硬件条件。

供电链路的设计重点并不在于追求高功率输出，而在于在较小板级面积内同时保证模拟前端、主控与无线链路的稳定供电。对于脑电采集系统而言，供电纹波、地参考波动以及蓝牙发射瞬态都有可能影响前端采集效果，因此在单板实现中将供电管理、电池接口和核心主控统一纳入设计，是实现整机稳定性的基础。此外，固件中还实现了基于按键与电源保持引脚的软关机逻辑，使设备具备更接近实际便携终端的工作方式，而非仅作为实验室内部的开发板使用。

\section{固件程序开发与流程设计}

在硬件电路基础上，系统固件负责将前端采集、主控处理、BLE 通信与状态管理整合为可运行的实时采集平台。本文固件基于 Zephyr RTOS 实现，采用模块化组织方式，将脑电采样、前端控制、数字滤波、无线传输以及扩展传感器处理等功能拆分到独立模块中。这样做的目的在于降低各功能之间的耦合度，并提高系统在后续功能增补和联调阶段的可维护性。

\subsection{固件核心工作流程}

系统上电后，固件首先完成主控基础外设初始化，包括 GPIO、ADC、蓝牙协议栈、NUS 服务以及脑电前端控制接口的配置；随后完成脑电前端芯片初始化与默认增益设置，并启动双通道采样模块。在采样任务正式运行后，系统以 `500\,\text{Hz}` 采样率驱动双通道采样流程，即每 `2\,\text{ms}` 触发一次采样调度。为了避免在定时器回调中执行耗时 ADC 读取操作，固件将真正的采样任务提交到独立工作队列中执行，从而降低实时调度被阻塞的风险。

在数据处理层面，采集到的双通道样本首先写入环形缓冲区，并附加设备侧时间戳；主循环随后从缓冲区中读取样本，执行数字滤波和状态处理，再根据当前 MTU 与批量打包策略构造 NUS 数据帧并通过 BLE 通知发送到上位机。同时，系统还持续监听来自上位机的文本命令，用于完成增益调整、滤波配置、缓存清理、状态查询和前端复位等控制任务。由此，系统形成了“周期采样 - 缓存入队 - 滤波处理 - 分帧发送 - 上位机回控”的闭环执行链路。

除脑电主链路外，固件中还包含独立的基础生理信号采集线程，用于读取扩展传感模块数据并通过文本消息向上位机回传结果。尽管这部分功能并非本章核心，但其存在说明该平台具备在低功耗嵌入式环境中同时管理多路采集任务与无线回传任务的能力。

\subsection{数字滤波与前端参数配置}

由于便携式脑电采集场景易受到工频干扰、低频漂移和高频噪声影响，本文在固件中实现了可配置的数字滤波策略。当前系统支持关闭滤波、工频陷波、低通滤波、组合滤波和带通滤波等多种模式，其中双通道默认采用组合滤波配置。相较于仅使用固定陷波器的单一实现方式，这种设计使系统能够根据不同实验任务和信号质量情况灵活切换处理方式。

从系统实现角度看，数字滤波的意义不仅在于改善波形观测效果，还在于将采集系统从“固定参数的示波终端”提升为“可配置的实验平台”。一方面，上位机可通过命令接口动态调整各通道滤波模式，使联调过程无需频繁修改固件；另一方面，前端增益也可通过命令接口在多个预设档位之间切换，从而适应不同被试个体差异和电极接触条件。脑电前端的导联脱落检测信号同样被纳入固件状态管理逻辑，并通过文本回执实时通知上位机。这种“前端状态感知 + 软件侧参数调节”的实现方式，有助于提高硬件系统在实际实验中的可用性。

\subsection{数据缓冲区与传输逻辑}

在无线双通道脑电采集中，采样任务与传输任务之间天然存在速率不一致问题。为解决这一问题，系统在固件中采用环形缓冲区作为采样与发送之间的解耦机制。当前实现中，缓冲区默认深度为 `1024` 个样本，最大支持 `2048` 个样本，并使用原子计数器记录已就绪样本数、总采样数以及缓冲区溢出次数。这样既能为上位机短时处理波动提供时间裕量，也便于后续对缓存状态和链路稳定性进行查询。

在传输侧，系统结合当前 BLE MTU 动态调整每帧携带的样本数量。连接建立后，设备会根据 MTU 更新结果重新计算批量发送的样本数上限，并为每个数据帧分配序列号。该策略相比逐点发送方式具有更高的有效负载利用率，更适合连续脑电流式数据的无线发送。与此同时，文本命令回执与二进制脑电帧采用同一 NUS 链路承载，但通过帧头魔数实现了解析层区分，从而在保持协议简洁的同时兼顾了实时数据流和状态控制流。总体而言，这种“环形缓冲 + 动态分帧 + 序列号跟踪”的实现机制，为双通道脑电在低功耗蓝牙链路中的稳定传输提供了软件支撑。

\section{系统联调验证与实验结果分析}

本章的重点在于硬件系统设计与实现，因此本节不将算法分类精度或大规模用户实验作为主要讨论对象，而是围绕目前已经完成的系统联调与基础验证实验，对该平台的闭环可用性进行说明。当前阶段已经能够确认的是：系统能够完成双通道脑电采样、前端参数配置、BLE 数据回传、导联状态上报以及上位机实时显示，说明该平台已经从板级设计推进到了可运行的系统原型阶段。基于这一原型，本文进一步开展了脑电采集有效性验证和标签链路验证，以支撑后续算法实验与系统集成工作。

\subsection{脑电采集有效性验证}

仅依靠波形能够被上位机连续显示，还不足以说明系统已经采集到具有研究意义的脑电信息。因此，本文进一步设计了闭眼/睁眼静息 Alpha 现象验证实验，用于考察当前双通道硬件平台采集结果是否具有基本的生理可解释性。实验中以上位机独立 Alpha 界面记录阶段化频谱结果，并采用 `8--13\,\text{Hz}` 频带功率占 `4--30\,\text{Hz}` 参考频带功率的比例作为主指标；在每组实验结束后自动输出睁眼阶段均值、闭眼阶段均值以及闭眼/睁眼比值，以避免仅依据单帧显示结果作出判断。

结合当前已经完成的两组代表性实验结果，可以观察到闭眼阶段的 Alpha 相对功率均高于睁眼阶段，如表\ref{tab:c3_alpha_validation}所示。其中，前额导联主结果的双通道综合指标由 `22.2563\%` 提升至 `25.7564\%`，闭眼/睁眼比值为 `1.1573`；顶部补充结果的对应比值为 `1.1424`。进一步考察各循环结果可知，前额导联三轮比值分别为 `1.129`、`1.143` 和 `1.211`，顶部导联三轮比值分别为 `1.131`、`1.270` 和 `1.034`，整体上均呈现“闭眼高于睁眼”的增强趋势。这说明当前系统不仅能够连续采集双通道脑电信号，而且能够在低通道条件下捕捉到具有一定重复性的频带变化特征。

从上述结果可以看出，当前系统在自动打标条件下不存在明显的丢标、阻塞或随机失稳问题。尽管设备侧实际标签间隔相较设定值普遍增加约 `34--40\,\text{ms}`，但这一偏差在不同节奏组中具有较强一致性，更可能来源于上位机定时器调度、事件循环处理与命令发送开销，而非设备端链路的不可控抖动。因此，现有结果足以支持“该系统已具备一般交互事件 EEG 标签写入能力”的结论，但并不支持将其表述为严格意义上的 ERP 级绝对锁时系统。

\subsection{无线传输稳定性分析}

在无线数据链路方面，系统通过“采样缓冲区解耦 + 序列号可观测协议 + 状态查询与回执”三层机制共同保障稳定运行。设备端使用环形缓冲区隔离采样与发送过程，并对缓冲区使用量和溢出次数进行统计；协议层在脑电帧中加入序列号与样本数量字段，上位机据此可以识别潜在丢包和乱序；控制层则支持状态查询、缓存清理、导联状态回报和错误回执，使联调阶段出现异常时能够较快定位问题。与单纯追求峰值吞吐的设计不同，该策略更强调连续采样场景下的可观测性与可调试性。

结合当前日常运行观察以及前述标签实验结果可以认为，在 `500\,\text{Hz}` 双通道连续采样和 BLE 持续连接条件下，系统短时运行总体稳定，未见明显持续性丢帧与异常中断。当前尚未完成按 `10\,\text{min}`、`30\,\text{min}`、`60\,\text{min}` 等固定时长组织的长时间量化统计，因此本节对无线链路稳定性的结论主要限于短时运行条件。

\subsection{供电与便携化实现分析}

供电与便携化设计是该系统区别于单纯桌面调试平台的重要特征。从硬件结构看，系统已经集成锂电池接口、Type-C 充电链路以及按键式上电/关机控制逻辑，说明其设计目标并非局限于实验桌面上的短时演示，而是面向可移动、可近体部署的脑电采集终端。将供电管理纳入系统整体设计后，脑电采集、无线传输和参数控制可以在脱离外部有线供电的条件下协同工作，这为后续进行便携式实验和边缘部署验证提供了前提。

目前，整机平均电流、不同工作模式功耗构成以及理论续航时间的量化测试尚未完成。因此，本节主要说明系统已经具备便携式实现所需的结构条件与供电逻辑，对最终功耗指标暂不作定量结论。

\section{本章小结}

本章围绕双通道脑电硬件平台的系统设计与实现展开论述，从总体架构、关键电路、固件流程以及联调验证四个层面说明了该平台的形成过程。通过 KS1092 前端、nRF52832 主控与 BLE 链路的协同设计，系统已经实现了从脑电采样、参数配置到上位机显示的基本闭环，并具备继续支撑后续算法实验和系统集成验证的工程基础。

结合本章后半部分的实验结果可以进一步看出，现有硬件平台已经不仅停留在“能够工作”的层面。闭眼/睁眼静息 Alpha 验证表明，系统在当前低通道导联条件下能够采集到具有一定重复性的频带变化；标签链路实验则说明，设备对交互事件标签具有较好的确认可靠性和节奏跟随能力。与此同时，无线传输长时稳定性以及整机功耗/续航仍有待后续补测。总体而言，本章完成了对硬件系统“已实现什么、已验证到什么程度、还需继续补充什么”的阶段性界定，为第 4 章算法设计和第 5 章系统级实验验证提供了硬件基础。
