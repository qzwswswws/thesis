# Algorithm Development Agent Brief

本文件不是普通讨论稿，而是 `MI_Algorithm_Workbench` 中算法开发 agent 的**最高优先级工作说明书**之一。  
agent 进入本工作区后，应先阅读本文件，再决定是否修改代码、运行实验或回写论文。

---

## 1. 任务定义

本课题的核心不是单纯“把分数刷高”，而是回答下面三个问题：

1. 在 `BCI Competition IV 2a` 上，从全通道降到低通道后，性能究竟如何退化？
2. 在低通道条件下，基于紧凑卷积前端与局部注意力的模型，能否比现有 baseline 更稳、更适合边缘部署？
3. 当真实硬件最终只支持极少通道时，应该如何把离线算法结果与工程展示逻辑连起来？

换句话说，本项目的主线不是“大模型刷榜”，而是：

`低通道约束 -> 算法补偿 -> 可部署性`

---

## 2. agent 必须接受的固定边界

### 2.1 数据与协议边界

- 主数据集固定为 `BCI Competition IV 2a`。
- 当前主线优先沿用已有可运行协议：
  - `A0xT` 训练
  - `A0xE` 测试
  - 单被试训练与测试
  - 使用训练集均值方差标准化测试集
- “全通道”输入数以**当前稳定代码的真实输入通道数**为准。
  - 若脚本最终确认是 `22` 导，则后续退化写成 `22 -> 8 -> 3 -> 2`
  - 若脚本最终确认是 `21` 导，则后续统一写成 `21 -> 8 -> 3 -> 2`
- 在协议冻结前，不允许同时修改：
  - 数据划分
  - 预处理
  - 通道数
  - 模型结构
  - 评价指标

### 2.2 研究边界

- 不优先切换到新数据集。
- 不优先转向交互系统、人机共适应、在线闭环等新问题。
- 不允许只在测试阶段对全通道模型做 masking 然后把它当成低通道结果。
- 不允许把尚未核验的部署数字直接写入论文正文。

### 2.3 工程边界

- 代码主线优先基于 `baselines/` 现有可运行脚本修补，不从零重写框架。
- 数据路径应通过 `BCI2A_DATA_ROOT` 显式声明，不再新增绝对路径硬编码。
- 新日志、图表、批量运行产物优先写入 `runs/`。
- 核心结果表优先汇总回 `results_summaries/`。

## 5. agent 可直接采用的知识结论

### 5.1 生理与通道选择知识

从 `chap02.tex` 和 `MI_EEG_sensorimotor_physiology_evidence_report.md` 可安全提炼出以下结论：

- 左右手运动想象的主要判别信息集中在中央感觉运动区。
- `C3`、`C4` 往往是左右手最重要的侧化信息来源。
- `Cz` 常作为补充信息源，而不是必然主通道。
- 因此，低通道路线优先选择：
  - `8` 导：围绕中央区的稀疏扩展
  - `3` 导：`C3, Cz, C4`
  - `2` 导：`C3, C4`
- 但不能把文献写成“C3/C4 永远足够”，因为跨被试差异是真实存在的。

### 5.2 频带与时序知识

- `mu` 与 `beta` 不是完全同一种空间分布。
- 不同受试者的最佳时序窗口和基线位置可能不同。
- 因此，低通道建模时不应只靠粗暴缩通道，更应保留足够的时间建模能力。

对 agent 的含义是：

- 低通道模型仍应优先保留**时间结构建模**
- 不建议把模型退化成过于浅层的静态特征分类器
- 更合理的方向是：`紧凑卷积前端 + 小型局部注意力块`

### 5.3 高效注意力知识

从 `Efficient_Transformers_for_EEG_real_time_inference_report.txt` 可提炼出以下结论：

- vanilla global self-attention 对 EEG 边缘推理不友好。
- EEG 领域当前更可靠的高效路线，不是“追求最纯粹的线性注意力公式”，而是：
  - 局部窗口注意力
  - 分层/滑窗聚合
  - 卷积前端先压 token，再做小型 Transformer
- compact backbone 的重要性不低于 attention 形式本身。

因此，agent 在设计 `EdgeMIFormer` 时，优先级应是：

1. 控制 token 数
2. 控制模型深度与头数
3. 使用局部窗口或近邻注意力
4. 先保证低通道稳定性，再谈更复杂的全局信息交互

### 5.4 部署知识

从 `Edge_deployment_evidence_for_RK3568_and_temporal_models.md` 可提炼出以下结论：

- MI-EEG 的边缘部署可行性本身已经有较强证据。
- 最强部署证据主要来自：
  - 紧凑 CNN
  - tiny transformer
  - MCU / PULP / FPGA 平台
- RK3568 作为目标平台是合理的，但当前文献对“MI EEG + transformer + RK3568”的直接证据仍不足。

因此，agent 需要遵守：

- 先做离线实验稳定闭环
- 再补复杂度与导出
- RK3568 的论文表述必须谨慎，避免写成“文献已充分证明”

### 5.5 模型压缩、限制约束与跨通道知识蒸馏

参考 `TinyML on MCU` 与 `Similarity-Keeping Knowledge Distillation` 等最新文献，提炼出可用于第 4/5 章前沿约束的论据：

- **极限部署约束**：针对穿戴式 MCU（与 RK3568 定位相似），不能为了 Accuracy 无限扩大模型。Data Marshaling（数据搬运开销）是 Transformer 最大软肋。
- **时间注意力主导地位**：由于通道极少，纯空间交互意义不大。`EEGformer` 等最新论文同样砍掉了空间结构，利用纯时序局部特征取得了 20% 的降迟缓优化。这为 `EdgeMIFormer` 收拢架构、放弃复杂大算子提供了合理背书。
- **降通道痛点可以被“跨通道蒸馏”解决**：前沿证明：既然 `22-channel` Baseline 可以跑到极高的准确率（85%），那么在硬件被压缩到 `2-channel` 性能断崖时（剩 50%），**可以让 22 导强模型充当“导师 (Teacher)”，将提取到的精细高维空间特征“蒸馏（Knowledge Distillation）”给 2 通道模型（Student）。** 

在此洞察下，遇到 2通道/4分类 崩盘问题时，不应立刻丢盔卸甲退让二分类，而应将知识蒸馏作为一剂强力算法补偿药剂！

## 6. 推荐主线：理论可行度最高的开发方案

### 6.1 总体策略

不建议在当前阶段走“双线并行”，也不建议直接切换到天然 2 通道数据集。

理论可行度最高、论文风险最低的主线是：

`同一数据集上的 full-channel baseline -> 4分类通道退化 -> 低通道下的任务重定义 -> EdgeMIFormer 针对性优化`

这条路线同时满足：

- 学术可比性强
- 变量控制清晰
- 与论文第 4/5 章高度一致
- 能把最终工程展示自然过渡到 2 通道场景

### 6.2 为什么不先换数据集

如果直接换到天然 2 通道/2 分类数据集，会引入不可控新变量：

- 数据质量变化
- 任务难度变化
- 被试分布变化
- 通道定义变化

这会削弱论文中“算法适配低通道硬件”的核心贡献，因为评审会质疑：

`究竟是算法变好了，还是数据集更容易了？`

所以，agent 在当前阶段应默认：

- **不换数据集**
- **不做双主线**
- **先把同数据集、同协议下的退化证据做完整**

## 7. 分阶段执行计划

### 阶段 A：协议冻结

目标：

- 最终确认 full-channel 的真实输入通道数
- 冻结当前 baseline 主入口
- 固化评价口径

最低完成标准：

- 明确记录：
  - full-channel 到底是 `21` 还是 `22`
  - 数据划分方式
  - 标准化方式
  - 当前 baseline 入口脚本
  - 当前 baseline 摘要文件

### 阶段 B：full-channel baseline 锁定

目标：

- 把 `window_size = 8` 这条已跑通结果确认为当前 full-channel baseline

建议动作：

- 不继续在 full-channel 上无限扫超参
- 只在必要时补 1 次复核
- 优先把 baseline 结果整理成可回写论文的规范表

原因：

- 你已经有一条新的 9 被试结果闭环
- 继续留在 full-channel 刷分，对论文边际收益不高

### 阶段 C：4分类通道退化主实验

目标：

- 在**同一任务**下证明低通道带来的信息损失

推荐顺序：

1. `full-channel`
2. `8` 导
3. `3` 导
4. `2` 导

固定要求：

- 每个通道配置独立训练
- 同一评价口径
- 保存实际通道列表

这是当前最重要的科学主线，因为它直接回答：

`低通道到底损失了多少？`

### 阶段 D：跨通道知识蒸馏主攻 -> 二分类任务重定义（保底）

触发条件：

- 当 `2` 导 `4` 分类结果明显崩盘，且普通网络继续结构优化收益低时，自动切入本阶段

目标分支：

- **首发主攻 D.1（知识蒸馏挽救）**：
  引入 **Teacher** 模型（阶段B确定好的全通道满分基线），教导 **Student** 模型（仅使用2通道的基线或 EdgeMIFormer）。利用 Similarity-Keeping等蒸馏技术将高维空间隐式特征注入双通道网络。若能将 4分类准确率由 50% 拉回可用区间（例如 65%~70%+），则学术成就极高，大获成功。
- **任务重定义 D.2（工程保底）**：
  若蒸馏提升有限，或出于部署严苛妥协考虑，在保留 `BCI 2a` 同一数据集大前提不变的情况下，将数据精准过滤重组为 `左手 vs 右手`（二分类），用 `EdgeMIFormer` 结合 `C3/C4` 攻破 80%+，确保最终硬件展示的高成功率闭环。

理论依据与价值：

- “核心算法+知识蒸馏找回高维空间特质”在学术界比单纯减法更具技术含金量，极大丰富第四章和第五章的实验章节厚度。
- 退保底到二分类符合 `C3/C4` 特征侧化规律，天然切合轮椅控制等落地应用原型。

### 阶段 E：`EdgeMIFormer` 候选模型介入

目标：

- 让本文模型不只存在于结构描述里，而能在关键设置下给出定位

优先顺序：

1. full-channel
2. `3` 导
3. `2` 导 `2` 分类

最可行的结构方向：

- 保留紧凑卷积前端
- 在 token 数受控后接局部窗口注意力
- 避免深层、宽头、重型全局注意力
- 优先用小模型证明“低通道下的精度-复杂度折中”

不建议：

- 一开始就用 `EdgeMIFormer` 全面替换 baseline
- 在低通道结果未稳定前，叠加太多新模块

### 阶段 F：复杂度与部署补充

目标：

- 在主实验稳定后补论文第 4/5 章需要的部署证据

顺序：

1. 参数量
2. 模型大小
3. FLOPs / MACs
4. ONNX 导出
5. 量化 / RKNN

这是尾部补充，不是主线起点。

## 8. 对算法 agent 最有用的代码层策略

### 8.1 先做“可配置筛选”，不要做散乱 if/else

agent 应优先把数据筛选能力做成参数化能力，而不是临时魔改：

- `selected_channels`
- `selected_classes`
- `remap_labels`
- `task_name`

推荐实现方式：

- 在数据加载阶段统一完成：
  - 类别过滤
  - 标签重映射
  - 通道索引选择
- 不要把类别过滤散落到训练循环中

### 8.2 先让 baseline 支持低通道与二分类，再谈新模型

优先顺序应是：

1. baseline 数据管线支持：
   - 全通道 / 8 / 3 / 2 导
   - 4 分类 / 2 分类
2. baseline 先跑通
3. 再迁移同样机制到 `EdgeMIFormer`

### 8.3 所有结果必须附带协议元信息

每轮实验至少记录：

- 入口脚本
- 通道列表
- 类别列表
- 是否重映射标签
- seed
- 结果文件位置

## 9. 决策门槛

agent 可以按以下门槛做推进判断：

### 可以继续推进 4 分类低通道

- `3` 导结果虽有下降，但仍具有分析意义
- `2` 导结果虽差，但能稳定复现同一趋势

### 应停止在 2 导 4 分类上无休止打磨

出现以下情况之一即可考虑转入二分类：

- `2` 导 `4` 分类结果长期停留在低水平
- 结果波动远大于结构改动带来的收益
- 优化成本明显高于论文收益

### 才值得进入部署补充

- baseline 已冻结
- 低通道退化曲线已形成
- `EdgeMIFormer` 至少在一个关键设置下有结果

---

## 10. 明确禁止的路线

- 禁止直接换数据集来“美化”低通道结果
- 禁止在协议尚未锁定时同步改动多个大变量
- 禁止把论文中已有的理想数字当成当前已验证事实
- 禁止一边做低通道主线，一边又把主精力切去做交互系统
- 禁止在没有 full-channel 对照的情况下直接宣称低通道方法有效

---

## 11. agent 的立即行动清单

如果这是一个新的算法开发会话，请按以下顺序执行：

1. 核验当前 baseline 脚本的真实 full-channel 通道数。
2. 将 baseline 的数据加载逻辑改造成支持：
   - 通道筛选
   - 类别筛选
   - 标签重映射
3. 先在 baseline 上做 `full-channel / 8 / 3 / 2` 的 4 分类 pilot。
4. 若 `2` 导 `4` 分类明显失稳，再切入 `左手 vs 右手` 二分类。
5. 只有在 baseline 路线跑顺后，才把相同协议迁移到 `EdgeMIFormer`。

## 12. 一句话版本（进阶版）

给算法开发 agent 的最简任务描述如下：

> 冻结 full-channel baseline 跑出 4 分类退化曲线；在 2 通道遭遇极大准确率瓶颈时，**切勿立马退守二分类，应先强势引入“22 通道到 2 通道的知识蒸馏（Knowledge Distillation）”做破局的算法增强核心**；若蒸馏成果达标则 4 分类告捷，若确实受限于物理上限则有尊严地退让至 `C3/C4` 左右手二分类，并结合 `EdgeMIFormer` 紧凑卷积前端+时序局部注意力榨干极限性能；全局绝不中途重写代码大换血、不切数据集，以这套严格的协议坚守到底构建完整的逻辑闭环。
